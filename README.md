# Buchbinder – the *Edelweiß* book project

This repository holds the manuscript of the novel ***Edelweiß*** as a set of
individual LibreOffice Writer files, together with **Buchbinder**, a small tool
that merges those files into a complete book and exports it to PDF, EPUB, HTML,
ODT or Markdown. It can also build a journal-only edition (*Wilhelms Tagebuch*).

- Project / format conventions: this file.
- How to run the tool and how it behaves: [docs/USAGE.md](docs/USAGE.md).

---

## 1. The source manuscript

The manuscript lives in [`data/edelweiss`](data/edelweiss). Each **section** of
the book is one `.odt` (LibreOffice Writer) file whose name encodes its position
and kind:

```
NNNN_Type_Title.odt
│ │  │    └── CamelCase version of the section title  (e.g. ListeDerWinter)
│ │  └─────── section type: "Story" or "Journal"
│ └────────── digits 3–4: the section index inside the chapter (01, 02, …)
└──────────── digits 1–2: the chapter number (01 … 20)
```

Examples:

| File name                              | Chapter | Section | Type    | Title              |
|----------------------------------------|:-------:|:-------:|---------|--------------------|
| `0101_Story_Morgenstimmung.odt`        | 1       | 1       | Story   | Morgenstimmung     |
| `0102_Journal_ListeDerWinter.odt`      | 1       | 2       | Journal | Liste der Winter   |
| `0203_Story_Bibliothek.odt`            | 2       | 3       | Story   | Bibliothek         |

The four-digit prefix defines the **reading order**. The first two digits group
sections into chapters.

### Story vs. Journal

A chapter is built from alternating **Story** and **Journal** sections:

- **Story** sections are the main narrative, set in an ordinary **serif** font
  (paragraph style *Textkörper [Story]*).
- **Journal** sections are diary-style passages, set in *italics*
  (paragraph style *Textkörper [Journal]*, which inherits from the Story style).

The alternation is the norm but not an absolute rule — a chapter may contain two
sections of the same kind in a row (e.g. chapter 16 has two Journal sections),
and some chapters consist of a single Story section.

### Section separators

Inside a chapter, consecutive sections are separated by a paragraph that
contains a single tilde **`~`** (centred). There is **no** tilde between
chapters — a new chapter simply starts with its own heading (chapters flow
continuously; only the front matter, i.e. title page and table of contents,
is followed by a page break).

Each chapter begins with the heading **“Kapitel *N*”** (where *N* is the chapter
number). In the merged book this heading is generated by the tool; the source
section files themselves do not carry it. A generated table of contents is
labelled **“Inhaltsverzeichnis”**.

### Special formatting inside the text

- **Poems** use a distinct layout: shorter lines, different indentation and
  different line spacing. They are realised with direct/automatic paragraph
  formatting plus manual line breaks rather than a dedicated named style.
- **Foreign-language passages** are marked with character styles whose names
  start with `eo_`. They change the typography so that the foreign words stand
  out — for example narrower/italic text, a different language tag, or a special
  font:

  | Character style | Purpose                                   | Effect                              |
  |-----------------|-------------------------------------------|-------------------------------------|
  | `eo_markiert`   | generic “marked” foreign text             | italic, condensed letter-spacing    |
  | `eo_fr`         | French                                    | language = fr-FR                    |
  | `eo_la`         | Latin                                     | language = la-VA                    |
  | `eo_lv`         | Latvian                                   | language = lv-LV                    |
  | `eo_en`         | English                                   | language = en-GB                    |
  | `eo_platt`      | Low German (Plattdeutsch)                 | condensed, italic                   |
  | `eo_jp`         | Japanese (romanised)                      | language tag, no spell-check        |
  | `eo_kanji`      | Japanese in Kanji/Kana                    | *Hiragino Mincho* font              |
  | `eo_q`          | special quotations                        | *Herculanum* font                   |
  | `eo_ain`        | Ainu                                      | inherits the Japanese style         |

All of this formatting is preserved automatically: Buchbinder carries every
paragraph and character style from the source files into the merged book, so
italics, poems and foreign-language passages keep their original appearance.

### What is *not* part of the book

The `data/edelweiss` folder also contains working material that is **not** merged
into the book, because it does not match the `NNNN_Type_Title.odt` pattern:

- `Sammelbindungen/` – the legacy LibreOffice master documents (see below).
- `Konzept/`, `Werkbank/`, `En/`, `papy/`, `Umschlagseiten/`, `Webseite/`, … –
  notes, drafts, cover pages and other auxiliary files.
- Loose files such as `Epilogue.odt`, `Retardation.odt`, `Konzept`/draft texts.

Buchbinder only ever looks at files that match the section naming pattern, so
these are ignored.

## 2. How the book used to be assembled

Previously the book was produced with LibreOffice **master documents**
(`.odm`) in [`data/edelweiss/Sammelbindungen`](data/edelweiss/Sammelbindungen):

- `Gesamt.odm` linked every section in order to form the complete book.
- `Kapitel_1.odm` … `Kapitel_20.odm` did the same per chapter.

A master document references the section files via links and inserts the `~`
separators between them. This works well for printing/PDF, but a master document
does **not** expand its linked sections when exported to EPUB (the export only
contains the master’s own paragraphs). Buchbinder was written to get a single,
reliable pipeline for **all** output formats.

---

## 3. How Buchbinder works

Buchbinder ([`tools/buchbinder.py`](tools/buchbinder.py)) reproduces the master
document idea, but instead of *linking* the sections it **merges** them into one
self-contained ODT:

1. **Discover & order** all `NNNN_Type_Title.odt` files and group them by
   chapter.
2. **Merge** the sections into a single ODT document:
   - every section’s automatic style names (`P1`, `T3`, `L1`, …) are renamed
     with a per-section prefix so they do not collide;
   - the named styles, fonts, page layout and master pages from the shared
     template are merged in once;
   - a generated “Kapitel *N*” heading is inserted at the start of each chapter,
     and `~` separators are inserted between the sections of a chapter;
   - optionally a title page and a clickable “Inhaltsverzeichnis” are added.
3. **Export** the merged ODT to the requested format(s) with LibreOffice in
   headless mode (`soffice --convert-to`). **Markdown** is generated directly
   in Python (LibreOffice has no usable Markdown export filter).

Because all styles travel with the content, the serif/italic distinction,
poems and foreign-language passages are reproduced faithfully.

Outputs are written under [`data/publish`](data/publish):

```
data/publish/
├── pdf/    Edelweiss.pdf      or  Kapitel_01.pdf, Kapitel_02.pdf, …
├── epub/   Edelweiss.epub     or  Kapitel_01.epub, …
├── html/   Edelweiss.html     or  Kapitel_01.html, …
├── md/     Edelweiss.md       or  Kapitel_01.md, …
└── odt/    Edelweiss.odt      (the merged source document)
```

With `--tagebuch` the journal-only edition is written as `Tagebuch.*`
(or `Tagebuch_Kapitel_NN.*` in chapters mode).

### Opening the EPUB

The generated EPUB is valid (`mimetype` first/stored, correct container). If
double-clicking it opens an XML/text editor instead of a reader, that is a
macOS **file-association** setting, not a problem with the file: right-click the
`.epub` → *Open With* → **Books** (or *Get Info* → *Open with* → choose Books →
*Change All…* to make it the default).

---

## 4. Requirements

- **Python 3** (standard library only).
- **LibreOffice** (the `soffice` binary) for any format other than `odt`.
  On macOS this is typically
  `/Applications/LibreOffice.app/Contents/MacOS/soffice`; Buchbinder finds it
  automatically or you can point at it with `--soffice`.

See [docs/USAGE.md](docs/USAGE.md) for the full command-line reference.
